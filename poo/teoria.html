1 ¿Qué es la Programación Orientada a Objetos?

Del código espagueti al código organizado

😞 Código procedural (spaghetti)
// 500 líneas en un archivo...
$nombre_guerrero = 'Aragorn';
$vida_guerrero   = 150;
$fuerza_guerrero = 30;
$nombre_mago     = 'Gandalf';
$vida_mago       = 80;
$mana_mago       = 100;

function atacar_guerrero($fuerza, &$vida_obj) {
    $vida_obj -= $fuerza + rand(0,10);
}
function atacar_mago($fuerza, &$mana, &$vida_obj) {
    // diferente lógica mezclada aquí...
}
Datos y funciones dispersos. Difícil de mantener.
✅ Código orientado a objetos
// Cada personaje es un OBJETO autocontenido
$guerrero = new Guerrero('Aragorn');
$mago     = new Mago('Gandalf');

// Cada objeto sabe cómo atacar
$guerrero->atacar($mago);
$mago->atacar($guerrero);

// Los datos están DENTRO del objeto
echo $mago->getVida();  // acceso controlado
echo $guerrero;        // __toString
Datos y comportamiento juntos. Legible y extensible.
Definición: La POO es un paradigma de programación que organiza el código en objetos — entidades que encapsulan estado (datos) y comportamiento (métodos). Una clase es el molde; un objeto es la instancia real.

2 Clase y Objeto — el molde y la instancia

La clase es el plano, el objeto es la casa construida

📋
Clase = Molde
Define la ESTRUCTURA. No ocupa memoria de "datos".
class Perro { ... }
↓ new
🐕
Objeto = Instancia
Un perro concreto con sus propios datos.
$firulais = new Perro('Firulais')
// ── DEFINIR la clase (el molde) ──────────────
class Perro
{
    // Propiedades: características del perro
    public string $nombre;
    public string $raza;
    public int    $edad;

    // Método: comportamiento del perro
    public function ladrar(): string
    {
        return $this->nombre . ' dice: ¡Guau!';
    }
}

// ── INSTANCIAR objetos (construir casas con el molde)
$firulais  = new Perro();
$firulais->nombre = 'Firulais';
$firulais->raza   = 'Labrador';
$firulais->edad   = 3;

$rex       = new Perro();
$rex->nombre = 'Rex';
$rex->raza   = 'Pastor Alemán';
$rex->edad   = 5;

// Cada objeto es INDEPENDIENTE
echo $firulais->ladrar(); // → Firulais dice: ¡Guau!
echo $rex->ladrar();      // → Rex dice: ¡Guau!

// $this → hace referencia al propio objeto

3 Constructor — inicializar el objeto

__construct() se llama automáticamente al hacer new

Constructor clásico
class Producto
{
    public string $nombre;
    public float  $precio;

    // __construct se llama al hacer "new Producto(...)"
    public function __construct(string $nombre, float $precio)
    {
        $this->nombre = $nombre;
        $this->precio = $precio;
    }
}

$laptop = new Producto('Laptop', 2899000);
echo $laptop->nombre; // → Laptop
Constructor Promotion (PHP 8) ✨
// PHP 8: el modificador en el parámetro
// crea la propiedad Y la asigna automáticamente
class Producto
{
    public function __construct(
        public  string $nombre,
        public  float  $precio,
        private int    $stock = 0   // valor por defecto
    ) {}
    // ¡Sin código extra! PHP lo hace automáticamente.
}

$laptop = new Producto('Laptop', 2899000, 10);
echo $laptop->nombre; // → Laptop
Tip: Usa Constructor Promotion en proyectos modernos (PHP 8+). Es más conciso y elimina código repetitivo.

4 Encapsulamiento — controlar el acceso

public, protected, private

ModificadorDesde la misma claseDesde una clase hijaDesde fuera
public ✓✓✓
protected✓✓✗
private ✓✗✗
 
class CuentaBancaria
{
    private float $saldo = 0;       // nadie accede directamente
    public  string $titular;         // todos pueden leerlo

    public function depositar(float $monto): void
    {
        if ($monto <= 0) throw new InvalidArgumentException('Monto inválido');
        $this->saldo += $monto;  // solo la clase modifica $saldo
    }

    public function getSaldo(): float   // getter: solo lectura
    {
        return $this->saldo;
    }
}

$cuenta = new CuentaBancaria();
$cuenta->depositar(1000);
echo $cuenta->getSaldo();    // → 1000  ✓
// $cuenta->saldo = 999999;  ← ERROR: propiedad privada

5 Herencia — reutilizar y extender

extends · parent:: · sobrescribir métodos

class Animal
{
    public function __construct(protected string $nombre) {}

    public function comer(): string
    {
        return $this->nombre . ' está comiendo.';
    }

    public function hacerSonido(): string
    {
        return '...';   // genérico, las hijas lo sobreescriben
    }
}

// Perro HEREDA todo de Animal y AGREGA / SOBREESCRIBE
class Perro extends Animal
{
    public function __construct(string $nombre, private string $raza)
    {
        parent::__construct($nombre);  // llama al constructor del padre
    }

    // Sobreescribir un método del padre
    public function hacerSonido(): string
    {
        return $this->nombre . ' (' . $this->raza . '): ¡Guau!';
    }

    // Método propio (solo Perro lo tiene)
    public function buscarPalo(): string
    {
        return $this->nombre . ' trae el palo 🦴';
    }
}

$perro = new Perro('Rex', 'Labrador');
echo $perro->comer();       // heredado de Animal → "Rex está comiendo."
echo $perro->hacerSonido(); // sobreescrito → "Rex (Labrador): ¡Guau!"
echo $perro->buscarPalo();  // solo Perro → "Rex trae el palo 🦴"

6 Clase Abstracta — el molde incompleto

abstract class · abstract method · no instanciable directamente

// Una clase abstracta:
//   ✓ Puede tener métodos concretos
//   ✓ Puede tener métodos abstractos (sin cuerpo)
//   ✗ NO se puede instanciar con "new"

abstract class Figura
{
    public function __construct(protected string $color) {}

    // Método ABSTRACTO: las hijas DEBEN implementarlo
    abstract public function area(): float;
    abstract public function perimetro(): float;

    // Método CONCRETO: compartido por todas las hijas
    public function describir(): string
    {
        return sprintf(
            'Figura %s: área=%.2f, perímetro=%.2f',
            $this->color, $this->area(), $this->perimetro()
        );
    }
}

class Circulo extends Figura
{
    public function __construct(string $color, private float $radio)
    {
        parent::__construct($color);
    }

    public function area(): float      { return M_PI * $this->radio ** 2; }
    public function perimetro(): float { return 2 * M_PI * $this->radio; }
}

class Rectangulo extends Figura
{
    public function __construct(string $color,
        private float $ancho, private float $alto)
    {
        parent::__construct($color);
    }

    public function area(): float      { return $this->ancho * $this->alto; }
    public function perimetro(): float { return 2 * ($this->ancho + $this->alto); }
}

// $figura = new Figura('rojo');  ← ERROR: clase abstracta
$c = new Circulo('azul', 5);
echo $c->describir();  // método HEREDADO usa area() sobreescrita
¿Cuándo usar clase abstracta?
  • Cuando varias clases comparten código base
  • Cuando quieres OBLIGAR a las hijas a implementar ciertos métodos
  • Cuando el concepto padre no tiene sentido por sí solo (no tiene sentido un "Figura" genérica, sí un "Círculo")
Diferencia con Interface:
  • La clase abstracta puede tener código concreto
  • Solo se puede extender una clase abstracta
  • La interface solo declara firmas
  • Una clase puede implementar varias interfaces

7 Interface — el contrato

interface · implements · múltiples interfaces

// Una interface define un CONTRATO: qué métodos debe tener la clase.
// No importa CÓMO los implementa — solo que EXISTEN.

interface Exportable
{
    public function exportarCSV(): string;
    public function exportarJSON(): string;
}

interface Imprimible
{
    public function imprimir(): void;
}

// Una clase puede implementar VARIAS interfaces
class Reporte implements Exportable, Imprimible
{
    public function __construct(private array $datos) {}

    public function exportarCSV(): string
    {
        return implode(',', $this->datos);
    }

    public function exportarJSON(): string
    {
        return json_encode($this->datos);
    }

    public function imprimir(): void
    {
        echo 'Imprimiendo: ' . $this->exportarCSV();
    }
}

$r = new Reporte(['Ana', 'Carlos', 'María']);
echo $r->exportarJSON(); // → ["Ana","Carlos","María"]

// Type hinting con interface
function enviarAlServidor(Exportable $elemento): void
{
    echo $elemento->exportarJSON(); // funciona con cualquier Exportable
}

8 Polimorfismo — mismo mensaje, diferente respuesta

Griego: polys (muchos) + morphe (forma)

abstract class Empleado
{
    public function __construct(protected string $nombre) {}
    abstract public function calcularSalario(): float;
    public function getNombre(): string { return $this->nombre; }
}

class EmpleadoFijo extends Empleado
{
    public function __construct(string $nombre, private float $salarioBase) {
        parent::__construct($nombre);
    }
    public function calcularSalario(): float { return $this->salarioBase; }
}

class EmpleadoPorHoras extends Empleado
{
    public function __construct(string $nombre,
        private float $valorHora, private int $horas) {
        parent::__construct($nombre);
    }
    public function calcularSalario(): float { return $this->valorHora * $this->horas; }
}

// POLIMORFISMO: todos son Empleado → mismo método calcularSalario()
$nomina = [
    new EmpleadoFijo('Ana', 3500000),
    new EmpleadoPorHoras('Carlos', 25000, 160),
    new EmpleadoFijo('María', 4200000),
];

$totalNomina = 0;
foreach ($nomina as $emp) {
    $salario = $emp->calcularSalario(); // PHP decide cuál usar según el tipo real
    echo $emp->getNombre() . ': $' . $salario;
    $totalNomina += $salario;
}
echo 'Total nómina: $' . $totalNomina;

9 Propiedades y métodos estáticos

static · self:: · Clase::método() sin instanciar

class Contador
{
    // Pertenece a la CLASE, no a una instancia
    private static int $instancias = 0;

    public function __construct()
    {
        self::$instancias++;   // self:: para acceder a static en la misma clase
    }

    public static function getInstancias(): int
    {
        return self::$instancias;
    }

    // Método utilitario estático (no necesita instancia)
    public static function formatearPrecio(float $precio): string
    {
        return '$' . number_format($precio, 0, ',', '.');
    }
}

$a = new Contador();
$b = new Contador();
$c = new Contador();

echo Contador::getInstancias();     // → 3  (sin necesidad de objeto)
echo Contador::formatearPrecio(1500000); // → $1.500.000

10 Traits — reutilizar código horizontal

Como un "mixin": agrega métodos a cualquier clase sin herencia

// Un trait define métodos reutilizables
// Se "inyectan" en la clase con "use"

trait Timestampable
{
    private string $creadoEn;
    private string $actualizadoEn;

    public function marcarCreado(): void
    {
        $this->creadoEn = date('Y-m-d H:i:s');
    }

    public function getCreadoEn(): string  { return $this->creadoEn; }
}

trait Loggable
{
    private array $log = [];

    public function registrar(string $msg): void
    {
        $this->log[] = date('H:i:s') . ': ' . $msg;
    }

    public function getLog(): array { return $this->log; }
}

class Usuario
{
    // ¡Usa AMBOS traits! Imposible con herencia múltiple.
    use Timestampable, Loggable;

    public function __construct(public string $nombre)
    {
        $this->marcarCreado();
        $this->registrar('Usuario creado: ' . $nombre);
    }
}

$u = new Usuario('Ana García');
echo $u->getCreadoEn();         // del Timestampable
print_r($u->getLog());          // del Loggable

11 Magic Methods — la magia de PHP

Métodos que PHP llama automáticamente en ciertos eventos

class Coleccion
{
    private array $items = [];

    // __construct: al hacer new Coleccion()
    public function __construct(array $items = []) {
        $this->items = $items;
    }

    // __toString: al hacer echo $coleccion o (string)$coleccion
    public function __toString(): string {
        return 'Colección[' . count($this->items) . ' elementos]';
    }

    // __get: al leer una propiedad que no existe
    public function __get(string $nombre): mixed {
        return $this->items[$nombre] ?? null;
    }

    // __set: al escribir una propiedad que no existe
    public function __set(string $nombre, mixed $valor): void {
        $this->items[$nombre] = $valor;
    }

    // __isset: al hacer isset($coleccion->algo)
    public function __isset(string $nombre): bool {
        return isset($this->items[$nombre]);
    }

    // __destruct: al destruir el objeto (fin de script o unset())
    public function __destruct() {
        // Bueno para cerrar conexiones, limpiar recursos...
    }
}

$c = new Coleccion(['color' => 'azul']);
echo $c;               // → "Colección[1 elementos]"  via __toString
echo $c->color;        // → "azul"  via __get
$c->forma = 'cuadrado'; // via __set
echo isset($c->forma); // → true  via __isset

__get / __set

12 PHP 8 — Novedades en POO

Constructor Promotion · Readonly · Enums · Named Arguments

Readonly Properties (PHP 8.1)
class Punto
{
    public function __construct(
        public readonly float $x,
        public readonly float $y
    ) {}
}

$p = new Punto(3.0, 4.0);
echo $p->x;    // → 3.0  ✓
// $p->x = 5; ← ERROR: readonly no se puede modificar
Enums (PHP 8.1)
enum Estado
{
    case Activo;
    case Inactivo;
    case Suspendido;
}

enum Color: string   // Backed Enum
{
    case Rojo  = 'red';
    case Azul  = 'blue';
    case Verde = 'green';
}

$e = Estado::Activo;
$c = Color::Rojo;
echo $c->value;  // → 'red'
Named Arguments (PHP 8.0)
function crearPersonaje(
    string $nombre,
    int    $vida   = 100,
    string $clase  = 'guerrero'
): array {
    return compact('nombre','vida','clase');
}

// Named: puedes omitir parámetros intermedios
crearPersonaje(nombre: 'Ana', clase: 'mago');
// → ['nombre'=>'Ana', 'vida'=>100, 'clase'=>'mago']
Match Expression (PHP 8.0)
$tipo = 'guerrero';

// match es más seguro que switch
$clase = match($tipo) {
    'guerrero' => new Guerrero('Aragorn'),
    'mago'     => new Mago('Gandalf'),
    'arquero'  => new Arquero('Legolas'),
    default    => throw new ValueError('Clase desconocida'),
};
