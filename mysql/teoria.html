1 El problema: los arrays se olvidan todo

¿Por qué necesitamos una base de datos?

😞 Con arrays en PHP
// Cada vez que recargas la página...
$contactos = [
    ['nombre' => 'Ana García', 'email' => 'ana@mail.com'],
    ['nombre' => 'Carlos',    'email' => 'carlos@mail.com'],
];

// Si el usuario crea un nuevo contacto:
$contactos[] = ['nombre' => 'Nuevo', ...];

// Al recargar la página...
// ¡EL CONTACTO NUEVO DESAPARECIÓ! 💨
// Los arrays viven solo en la petición actual.
Problema: PHP no tiene memoria entre peticiones. Cada request empieza desde cero.
✅ Con MySQL
// Los datos viven en el servidor MySQL
// y persisten entre peticiones:

$pdo = DB::conectar();

// Insertar un contacto nuevo:
$stmt = $pdo->prepare(
    'INSERT INTO contactos (nombre, email)
     VALUES (:nombre, :email)'
);
$stmt->execute([':nombre' => 'Nuevo', ...]);

// Al recargar la página...
// ¡El contacto SIGUE AHÍ! 🎉
Solución: MySQL guarda los datos en disco. Persisten entre peticiones, reinicios del servidor, etc.

Las 4 operaciones SQL fundamentales (CRUD)
-- Leer todos los registros
SELECT * FROM productos;

-- Leer con condición
SELECT nombre, precio
FROM productos
WHERE categoria = 'Periféricos'
ORDER BY precio DESC;

INSERT INTO productos (nombre, precio)

UPDATE productos
SET precio = 75000
WHERE id = 1;

-- Eliminar un registro
DELETE FROM productos
WHERE id = 1;

-- Contar registros
SELECT COUNT(*) FROM productos;
-- → 10

3 PDO vs MySQLi — ¿Cuál usar?

PHP ofrece dos formas de conectarse a MySQL. Siempre elegimos PDO.

Característica PDO ✅ MySQLi
Soporta múltiples BD ✓ MySQL, PostgreSQL, SQLite… ✗ Solo MySQL
Prepared Statements ✓ Nativo y limpio ⚠ Verboso
API orientada a objetos ✓ Solo OO Procedural y OO mezclados
Manejo de errores ✓ Excepciones (try/catch) ⚠ Códigos de error
Parámetros con nombres ✓ :nombre ✗ Solo posicionales (?)
Conclusión: Usa PDO siempre. Si algún día migras de MySQL a PostgreSQL, solo cambias el DSN — el resto del código queda igual.

4 Tu primera conexión PDO

El hola mundo de las bases de datos

<?php
// ── Paso 1: DSN (Data Source Name) ────────────────────────────
// Formato: 'driver:host=HOST;dbname=BD;charset=CHARSET'
$dsn = 'mysql:host=localhost;dbname=sena_tienda;charset=utf8mb4';

// ── Paso 2: Opciones recomendadas ─────────────────────────────
$opciones = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,  // lanza excepciones
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,         // arrays asociativos
    PDO::ATTR_EMULATE_PREPARES   => false,                      // prepared reales
];

// ── Paso 3: Crear la instancia PDO ─────────────────────────────
try {
    $pdo = new PDO($dsn, 'root', '', $opciones);
    echo 'Conectado a MySQL 🎉';
} catch (PDOException $e) {
    echo 'Error: ' . $e->getMessage();
    // En producción: log del error, NO mostrarlo al usuario
}
DSN
Data Source Name. Le dice a PDO qué driver usar, dónde conectarse y a qué BD.

mysql: driver
host= servidor
dbname= base de datos
charset=utf8mb4 tildes y emojis
Opciones clave
ERRMODE_EXCEPTION
Sin esto, los errores pasan silenciosamente. Con esto lanza PDOException.

FETCH_ASSOC
Las filas se devuelven como array asociativo: $fila['nombre'].

EMULATE_PREPARES = false
Prepared Statements reales, no simulados.
Errores comunes
Access denied
Usuario o contraseña incorrectos.

Unknown database
La BD no existe. Ejecuta primero setup.sql.

Connection refused
MySQL no está activo. Arranca XAMPP.

5 Clase Database — Patrón Singleton

Cómo reutilizar la conexión en toda la aplicación

Si en cada archivo PHP creas new PDO(...), tu app abrirá múltiples conexiones a MySQL. El patrón Singleton garantiza que se crea una sola vez y se reutiliza. 

<?php
// config/db.php
class DB
{
    private const HOST    = 'localhost';
    private const NOMBRE  = 'sena_tienda';
    private const USUARIO = 'root';
    private const CLAVE   = '';

    /** @var PDO|null  La instancia única */
    private static ?PDO $conexion = null;

    public static function conectar(): PDO
    {
        if (self::$conexion === null) {           // ¿primera vez?
            $dsn = 'mysql:host=' . self::HOST .
                   ';dbname='  . self::NOMBRE .
                   ';charset=utf8mb4';
            $opts = [
                PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
                PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
                PDO::ATTR_EMULATE_PREPARES   => false,
            ];
            self::$conexion = new PDO($dsn, self::USUARIO, self::CLAVE, $opts);
        }
        return self::$conexion;   // siempre devuelve la misma instancia
    }

    private function __construct() {}  // evita "new DB()"
    private function __clone()     {}  // evita clonar
}

// ── Uso desde cualquier archivo PHP ──────────────────────────
// require_once __DIR__ . '/config/db.php';
// $pdo = DB::conectar();

6 Leer datos — SELECT

Los métodos de fetch y cuándo usar cada uno

query() — Sin parámetros del usuario
$pdo = DB::conectar();

// Sin parámetros de usuario → query() está bien
$stmt = $pdo->query('SELECT * FROM productos ORDER BY nombre');

// fetchAll → array de todas las filas
$productos = $stmt->fetchAll();
// → [ ['id'=>1,'nombre'=>'Laptop',...], ['id'=>2,...] ]

// Recorrer con foreach
foreach ($productos as $p) {
    echo $p['nombre'] . ' → $' . $p['precio'];
}
fetch() y fetchColumn()
// fetch() → UNA sola fila (la primera)
$stmt = $pdo->prepare('SELECT * FROM productos WHERE id = ?');
$stmt->execute([1]);
$producto = $stmt->fetch();
// → ['id'=>1,'nombre'=>'Laptop',...] o false si no existe

if (!$producto) {
    echo 'No encontrado';
}

// fetchColumn() → solo el primer valor escalar
$total = $pdo->query('SELECT COUNT(*) FROM productos')
              ->fetchColumn();
// → 10

// fetchAll(PDO::FETCH_COLUMN) → array simple
$cats = $pdo->query('SELECT DISTINCT categoria FROM productos')
             ->fetchAll(PDO::FETCH_COLUMN);
// → ['Audio','Computadores','Monitores',...]
fetchAll()
Array con TODAS las filas. Usa para listas.
fetch()
Solo la primera fila. Usa para "ver uno".
fetchColumn()
Solo el primer valor. Usa con COUNT().

7 SQL Injection — el ataque más peligroso

Ocurre cuando un atacante inyecta código SQL en tu consulta. Puede leer, modificar o borrar toda tu base de datos.

❌ CÓDIGO VULNERABLE
// El usuario escribe en un formulario:
// $_GET['id'] = "1 OR 1=1"

$id = $_GET['id'];   // "1 OR 1=1"

// Concatenar directamente es PELIGROSO:
$sql = "SELECT * FROM productos WHERE id = $id";

// La consulta resultante es:
// SELECT * FROM productos WHERE id = 1 OR 1=1
//
// 1=1 siempre es verdadero → devuelve TODOS los productos
// El atacante accedió a datos que no debía ver.
Peor caso:
'; DROP TABLE productos; --
Esto elimina toda la tabla si el código es vulnerable.
✅ CÓDIGO SEGURO con PDO
// El usuario sigue escribiendo "1 OR 1=1"

$id = $_GET['id'];   // "1 OR 1=1"

// Con Prepared Statement:
$stmt = $pdo->prepare('SELECT * FROM productos WHERE id = ?');
$stmt->execute([$id]);

// PDO trata el valor "1 OR 1=1" como un string literal,
// no como código SQL. La consulta enviada es:
// SELECT * FROM productos WHERE id = '1 OR 1=1'
//
// → No devuelve nada (ningún id es "1 OR 1=1")
// ¡El ataque falla completamente!
Regla de oro: Cualquier valor que venga de $_GET, $_POST o de cualquier fuente externa DEBE usarse con Prepared Statements. Nunca concatenes variables en SQL.

8 Prepared Statements — la forma segura

prepare() + execute() — el dúo dinámico de PDO

Parámetros posicionales (?)
// El ? es el placeholder
$stmt = $pdo->prepare(
    'SELECT * FROM productos
      WHERE categoria = ?
      AND precio < ?'
);

// execute() recibe un array con los valores
// en el mismo orden que los ?
$stmt->execute(['Periféricos', 200000]);

$resultados = $stmt->fetchAll();
Parámetros nombrados (:nombre)
// Los :placeholders tienen nombre
$stmt = $pdo->prepare(
    'INSERT INTO contactos (nombre, email, categoria)
     VALUES (:nombre, :email, :categoria)'
);

// execute() recibe un array asociativo
$stmt->execute([
    ':nombre'    => 'Ana García',
    ':email'     => 'ana@mail.com',
    ':categoria' => 'Trabajo',
]);
bindValue() para LIMIT y OFFSET
// Para LIMIT y OFFSET necesitas bindValue con PDO::PARAM_INT
$stmt = $pdo->prepare(
    'SELECT * FROM productos LIMIT ? OFFSET ?'
);
$stmt->bindValue(1, 10, PDO::PARAM_INT);
$stmt->bindValue(2, 0,  PDO::PARAM_INT);
$stmt->execute();
rowCount() — filas afectadas
// Después de UPDATE o DELETE:
$stmt = $pdo->prepare('DELETE FROM productos WHERE id = ?');
$stmt->execute([5]);

// ¿Cuántas filas se eliminaron?
$afectadas = $stmt->rowCount();
// → 1 si existía, 0 si no existía

if ($afectadas === 0) {
    echo 'El producto no existía.';
}

9 INSERT — Crear registros

Patrón completo: formulario → validar → insertar → redirigir

<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    // 1. Recoger y limpiar
    $nombre = trim($_POST['nombre'] ?? '');
    $email  = trim($_POST['email']  ?? '');
    $precio = $_POST['precio'] ?? '';

    // 2. Validar
    $errores = [];
    if (empty($nombre))                                   $errores[] = 'Nombre obligatorio.';
    if (!filter_var($email, FILTER_VALIDATE_EMAIL))     $errores[] = 'Email inválido.';
    if (!is_numeric($precio) || $precio < 0)            $errores[] = 'Precio inválido.';

    // 3. Si no hay errores, INSERT
    if (empty($errores)) {
        try {
            $pdo  = DB::conectar();
            $stmt = $pdo->prepare(
                'INSERT INTO productos (nombre, email, precio)
                 VALUES (:nombre, :email, :precio)'
            );
            $stmt->execute([
                ':nombre' => $nombre,
                ':email'  => $email,
                ':precio' => (float)$precio,
            ]);

            // lastInsertId() devuelve el ID auto_increment creado
            $id = $pdo->lastInsertId();

            // 4. Patrón PRG: redirigir para evitar reenvío del form
            header("Location: ver.php?id=$id");
            exit;

        } catch (PDOException $e) {
            $errores[] = 'Error al guardar.';
        }
    }
}

10 UPDATE — Actualizar registros

Siempre incluir WHERE id = ? — sin WHERE actualizas TODA la tabla

$stmt = $pdo->prepare(
    'UPDATE productos
     SET nombre   = :nombre,
         precio   = :precio,
         categoria = :categoria
     WHERE id = :id'
);

$stmt->execute([
    ':nombre'    => 'Laptop Pro',
    ':precio'    => 3200000,
    ':categoria' => 'Computadores',
    ':id'        => 1,   // ← SIEMPRE especifica el ID
]);

// Verificar que se actualizó algo
if ($stmt->rowCount() === 0) {
    echo 'No se encontró el producto.';
}
⚠ Cuidado: UPDATE productos SET precio = 0 (sin WHERE) pone el precio de todos los productos a 0. Siempre filtra por ID.

11 DELETE — Eliminar registros

Con confirmación previa y verificación con rowCount()

// Siempre pide confirmación antes de eliminar
// Usa un formulario POST, no un simple enlace GET

if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_POST['confirmar'])) {

    $id = filter_input(INPUT_POST, 'id', FILTER_VALIDATE_INT);

    if ($id) {
        $stmt = $pdo->prepare('DELETE FROM productos WHERE id = ?');
        $stmt->execute([$id]);

        if ($stmt->rowCount() > 0) {
            header('Location: index.php?ok=delete');
            exit;
        }
    }
}

// El formulario de confirmación se ve así:

// <form method="POST">

//   <button name="confirmar" value="1">Sí, eliminar</button>
//   <a href="ver.php?id=5">Cancelar</a>
// </form>

12 Manejo de errores con try/catch

Cómo manejar PDOException correctamente

try {
    $pdo  = DB::conectar();
    $stmt = $pdo->prepare('SELECT * FROM productos WHERE id = ?');
    $stmt->execute([$id]);
    $producto = $stmt->fetch();

} catch (PDOException $e) {

    // ── En DESARROLLO: muestra el error completo ──────────────
    error_log('DB Error: ' . $e->getMessage());  // guarda en log
    echo 'Debug: ' . $e->getMessage();          // solo en desarrollo

    // ── En PRODUCCIÓN: mensaje genérico ──────────────────────
    // $error = 'Ocurrió un error. Intenta más tarde.';
    // No muestres $e->getMessage() al usuario final.
}

// Información útil en el objeto PDOException:
// $e->getMessage()  → descripción del error
// $e->getCode()     → código SQL (ej: 23000 = clave duplicada)
// $e->getFile()     → archivo donde ocurrió
// $e->getLine()     → línea donde ocurrió
Errores comunes de MySQL
CódigoCausa
1062Clave duplicada (UNIQUE)
1048Campo NOT NULL con valor nulo
1452Foreign key inválida
2002MySQL no está activo
1045Usuario/contraseña incorrectos
Detectar clave duplicada
try {
    $stmt->execute([...]);
} catch (PDOException $e) {
    if ($e->getCode() == '23000') {
        $error = 'El email ya está registrado.';
    } else {
        $error = 'Error de base de datos.';
    }
}

13 Buenas prácticas

Checklist para escribir código PHP + MySQL de calidad

✓ Siempre hacer
  • Usar Prepared Statements con cualquier input del usuario
  • Escapar HTML con htmlspecialchars() al mostrar datos
  • Envolver en try/catch cada llamada a la BD
  • Validar y castear parámetros GET/POST
  • Usar Patrón PRG después de INSERT/UPDATE/DELETE
  • Confirmar antes de eliminar
  • Usar utf8mb4 en el DSN y en la BD
✗ Nunca hacer
  • Concatenar variables en SQL: "WHERE id = $id"
  • Mostrar $e->getMessage() al usuario final
  • Guardar contraseñas en texto plano
  • Usar root sin contraseña en producción
  • UPDATE o DELETE sin WHERE
  • Mostrar datos de la BD sin htmlspecialchars()
  • Abrir múltiples conexiones en vez de reutilizar
