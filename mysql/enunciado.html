Reglas de seguridad obligatorias

✓ Siempre usar Prepared Statements
Cualquier consulta con parámetros del usuario ($_GET, $_POST) DEBE usar prepare() + execute().
✓ Escapar con htmlspecialchars()
Todo valor de la BD que se muestre en HTML debe pasar por htmlspecialchars() para evitar XSS.
✓ Validar IDs antes de usar
Usar filter_input(INPUT_GET, 'id', FILTER_VALIDATE_INT) y redirigir si el ID es inválido.
✓ Try/catch en todo acceso a BD
Envuelve CADA llamada a DB::conectar() y a $stmt->execute() en un bloque try { } catch (PDOException $e) { }.
